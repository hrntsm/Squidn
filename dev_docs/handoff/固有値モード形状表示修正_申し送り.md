# 固有値モード形状表示修正（縮約座標の誤読）・質量ランク過少検出修正 申し送り

作成日: 2026-07-23
対象コード: `crates/squid-n-solver/src/dynamic/eigen/mod.rs`、
`crates/squid-n-app/src/viewer/mod.rs`

## 1. 症状と原因

固有値解析のモード形状表示で、剛床のはずの同一階の節点が同一の水平変位に
ならず、特定の節点だけが動いて見えた。

原因は可視化側の座標系の誤読。`ModalResult::shapes` は拘束縮約
（剛床・剛リンク・MPC・支持固定）後の**独立自由度座標**（長さ
`Reducer::n_indep`）だが、ビューアの `ViewMode::Mode`
（`viewer/mod.rs`）はこれを「節点番号×6+方向」のグローバル自由度順として
直接読んでいた。縮約が 1 つでもある（実務モデルではほぼ常に該当。支点固定
だけでもズレる）と、先頭の節点に無関係な値が散り、残りはゼロになる。
これが「特定の節点のみ動く」表示の正体。

ソルバ内部の消費箇所（刺激係数 `compute_participation`、時刻歴の
モード減衰 `Damping::modal`、非線形時刻歴の初期モード）はいずれも縮約座標を
正しく前提としており、解析自体に誤りはなかった。

## 2. 修正内容

- `ModalResult` に `node_shapes: Vec<Vec<[f64; 6]>>` を追加。ソルバ側で
  各モードを `Reducer::expand_u` → `DofMap::active` 散布で節点×6成分へ展開して
  格納する（静的解析の `Analysis::expand_disp` と同一経路の
  `expand_node_shape`）。剛床スレーブにはマスターの剛体変位関係
  （面内並進＋回転）に従った値が入る。
- `shapes`（縮約座標）は従来のまま維持。時刻歴のモード減衰
  （`damping.rs::modal_c` が縮約次元と長さ厳密一致を要求）等の既存消費者を
  壊さないため。
- ビューアは `node_shapes.get(mode_idx)` を読むだけに変更し、
  `g = ni*6+d` の誤読ロジックを削除。

## 3. あわせて修正: 質量ランクの過少検出（並進・回転慣性のスケール差）

回帰テスト作成時に判明した固有値解析側の実バグ。剛床マスターに並進質量
（t オーダー）と回転慣性（t·mm²、並進の 10^6〜10^8 倍）を併せて与えた
モデルで、`gevd_jacobi` の質量ランク判定（射影質量行列 M̄ の固有値を
最大値との相対 1e-9 で切る）がスケール差により物理的に有意な並進方向を
「質量なし」と誤判定し、解けるはずの要求モード数で
「質量が有効な独立自由度が N 個しか見つかりませんでした」と失敗していた
（柱4本＋剛床・質量ランク3・要求3モードで再現。剛床で縮約後の独立自由度が
多いほど顕在化。ユーザー報告の「要求6モードに対し5個」エラーもこの症状と
整合する）。

第1段の修正は「射影行列を K̄ の対角で対称スケーリング（S=diag(1/√k̄ᵢᵢ)、
合同変換なので一般化固有値 θ は不変）してから M̃ の固有分解で質量ランクを
判定する」方式だった。M̃ の固有値は「柔性あたりの質量」（≈1/θ）という
スケール不変量になり、判定が単位系・基底スケールに依存しなくなる。

ただしこの「M̃ の固有分解で質量部分空間を分離してから解く」構造自体に、
反復ごとのランク判定が数値ノイズで揺らぐと「ランク落ち判定→ヌルベクトル
注入→K⁻¹ の冪乗反復による最低次モード方向への倒れ込み（平行化）→
再ランク落ち」という周期2のリミットサイクルに陥り NonConvergence になる
頑健性ギャップが残っていた（剛床マスターに実質量が乗った対称ポータル
フレームで顕在化。§4 の配線を入れて初めて表面化した）。最終的な実装は
**Bathe の一般化 Jacobi 法**（Finite Element Procedures §11.3.3）で、
K̄・M̄ を反復中のランク判定なしに正則な合同変換のみで同時対角化する。
質量ランク判定は対角化後の質量対角成分 m̂ᵢᵢ（≈1/θ、スケーリングにより
スケール不変）の相対値で最後に 1 回だけ行う。あわせて部分空間反復の
更新式を教科書どおりの y=K⁻¹·M·x に正した（従来は M を掛けない
y=K⁻¹·x だった）。

回帰テスト: `test_eigen_mass_rank_translation_rotation_scale_mix`
（`make_four_column_diaphragm_model`、修正前は「2個しか見つからない」で失敗）、
`test_eigen_subspace_matches_dense_ground_truth_q_lt_n`（q<n の部分空間反復を
dense 直接解と照合）、`test_generate_stories_action_reuses_rep_node_on_regenerate`
（squid-n-app。リミットサイクルの再現経路）。

## 4. 地震用重量→節点質量の接続（実装済み）

§3 までの調査で「`Node::mass` へ書き込む本番コードが存在せず、階自動生成の
地震用重量が 3D 固有値解析の質量に効かない」設計ギャップが判明し、
本件で接続を実装した。

- `Model::mass_method`（`squid_n_core::model::MassMethod`、serde default）が
  質量方式の単一情報源。**補正質点方式 CorrectedLumped（既定）**と
  **質点のみ方式 LumpedOnly** の 2 方式。
- 階自動生成（`story_gen/generate.rs::generate_stories_with_opts`）が剛床
  マスター節点へ質点質量を設定する:
  - 補正質点: net_i = max(0, w_i − w_self_i)（w_self_i は解析の質量行列に
    密度質量として計上される主架構部材・壁パネルの自重配分のみ。二次部材・
    雑壁・ダンパーは解析質量に入らないため控除しない）。
    mt = Σnet_i/g（Ux,Uy）、J = Σ(net_i/g)·r_i²（Rz、r_i は慣性力重心からの
    平面距離）。密度による自重との二重計上はこの控除で防ぐ。
  - 質点のみ: 控除なしの全量 W/g・J。`assemble_global_m` が
    `mass_method == LumpedOnly` のとき部材密度の要素質量をスキップする。
- 質量定義は固有値・時刻歴・SemiPrecise 周期算定の 3 者で共有
  （すべて `assemble_global_m` 経由のため自動的に一貫）。
- アプリ: `AnalysisSettings::mass_method` ＋固有値パネルの「質量方式」
  ComboBox。**設定は階の自動生成の実行時にモデルへ反映される**
  （`ApplyStories` が `model.mass_method` を設定、undo 対応）。

### 残課題・注意点

- MCP のジョブ入力には質量方式を公開していない（モデルの `mass_method` に
  従うのみ）。必要になったら `crates/squid-n-mcp` のジョブ入力を拡張する。
- 質量方式の設定を変えただけでは既存モデルは変わらない。階の自動生成を
  再実行して初めてマスター質量と `mass_method` が更新される（UI ヒントに明記済み）。
- 剛性率・偏心率の重心変位 `cog_horizontal_disp`
  （`squid-n-design-jp/src/secondary/stiffness_ratio.rs`）は質量加重平均を
  意図した設計で、これまで質量未設定のため単純平均へフォールバックしていた。
  マスターに質量が乗ったことで意図どおり「重心位置（マスター）の変位」を
  返すようになる（剛床では厳密に重心変位。挙動変化だが設計意図どおり）。
- 密度未設定の材料だけで組んだモデルでも、階を生成すればマスター質点で
  固有値解析が可能になる。階を生成しない・重量ゼロのモデルは従来どおり
  「質量がゼロです」等のエラーになる。
