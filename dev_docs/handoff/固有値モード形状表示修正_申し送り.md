# 固有値モード形状表示修正（縮約座標の誤読）・質量ランク過少検出修正 申し送り

作成日: 2026-07-23
対象コード: `crates/squid-n-solver/src/dynamic/eigen/mod.rs`、
`crates/squid-n-app/src/viewer/mod.rs`

## 1. 症状と原因

固有値解析のモード形状表示で、剛床のはずの同一階の節点が同一の水平変位に
ならず、特定の節点だけが動いて見えた。

原因は可視化側の座標系の誤読。`ModalResult::shapes` は拘束縮約
（剛床・剛リンク・MPC・支持固定）後の**独立自由度座標**（長さ
`Reducer::n_indep`）だが、ビューアの `ViewMode::Mode`
（`viewer/mod.rs`）はこれを「節点番号×6+方向」のグローバル自由度順として
直接読んでいた。縮約が 1 つでもある（実務モデルではほぼ常に該当。支点固定
だけでもズレる）と、先頭の節点に無関係な値が散り、残りはゼロになる。
これが「特定の節点のみ動く」表示の正体。

ソルバ内部の消費箇所（刺激係数 `compute_participation`、時刻歴の
モード減衰 `Damping::modal`、非線形時刻歴の初期モード）はいずれも縮約座標を
正しく前提としており、解析自体に誤りはなかった。

## 2. 修正内容

- `ModalResult` に `node_shapes: Vec<Vec<[f64; 6]>>` を追加。ソルバ側で
  各モードを `Reducer::expand_u` → `DofMap::active` 散布で節点×6成分へ展開して
  格納する（静的解析の `Analysis::expand_disp` と同一経路の
  `expand_node_shape`）。剛床スレーブにはマスターの剛体変位関係
  （面内並進＋回転）に従った値が入る。
- `shapes`（縮約座標）は従来のまま維持。時刻歴のモード減衰
  （`damping.rs::modal_c` が縮約次元と長さ厳密一致を要求）等の既存消費者を
  壊さないため。
- ビューアは `node_shapes.get(mode_idx)` を読むだけに変更し、
  `g = ni*6+d` の誤読ロジックを削除。

## 3. あわせて修正: 質量ランクの過少検出（並進・回転慣性のスケール差）

回帰テスト作成時に判明した固有値解析側の実バグ。剛床マスターに並進質量
（t オーダー）と回転慣性（t·mm²、並進の 10^6〜10^8 倍）を併せて与えた
モデルで、`gevd_jacobi` の質量ランク判定（射影質量行列 M̄ の固有値を
最大値との相対 1e-9 で切る）がスケール差により物理的に有意な並進方向を
「質量なし」と誤判定し、解けるはずの要求モード数で
「質量が有効な独立自由度が N 個しか見つかりませんでした」と失敗していた
（柱4本＋剛床・質量ランク3・要求3モードで再現。剛床で縮約後の独立自由度が
多いほど顕在化。ユーザー報告の「要求6モードに対し5個」エラーもこの症状と
整合する）。

修正: 射影行列を K̄ の対角で対称スケーリング（S=diag(1/√k̄ᵢᵢ)、
K̃=S·K̄·S、M̃=S·M̄·S。合同変換なので一般化固有値 θ は不変）してから
M̃ の固有分解で質量ランクを判定する。M̃ の固有値は「柔性あたりの質量」
（≈1/θ）というスケール不変量になり、単位系・基底スケールに依存しない。
固有ベクトルは z=S·z̃ で逆変換し、質量方向は M̄ 正規直交を維持、
質量ゼロ方向は単位ノルムへ正規化（スケール不揃いのまま反復すると
部分空間基底が退化し NonConvergence になるため。実際に正規化なしでは
既存テスト2件が収束しなくなることを確認済み）。

回帰テスト: `test_eigen_mass_rank_translation_rotation_scale_mix`
（`make_four_column_diaphragm_model`、修正前は「2個しか見つからない」で失敗）。

既知の限界（修正前から存在、今回の修正による回帰ではない）: 逆方向の
極端なスケール比（並進質量が回転慣性の 10^14 倍など、実建物では
起こらない単位バランス）では依然として質量ランクを過少検出し得る。
対応するなら S に M̄ 対角も加味したスケーリングが候補（レビュー時の検証で
確認。回転慣性≫並進質量という実務方向は本修正で解決済み）。

## 4. 関連する未解決課題: 固有値解析の質量が地震用重量と未配線

同じ調査で判明した設計ギャップ（今回の修正範囲外）:

- 固有値解析の質量源は「材料密度による整合質量」と「`Node::mass`
  （節点質量）」の 2 つのみ（`assemble_global_m`）。
- ところが `Node::mass` へ書き込む本番コードは全クレートに存在しない
  （`mass: Some(..)` はテストフィクスチャのみ）。階自動生成
  （`story_gen/generate.rs`）は地震用重量を `Story::seismic_weight` /
  `DiaphragmDef::weight` に持つが、剛床マスター節点の `Node::mass` へは
  変換していない。`seismic_weight` が効くのは EX/EY の Ai 分布同期と
  串団子モデル（`lumped_mass`）だけで、3D 固有値解析には効かない。
- したがって密度未設定の材料でモデルを組むと質量がほぼゼロになり、
  §3 の修正後であっても（真の質量ランク不足として）
  「質量が有効な独立自由度が N 個しか見つかりませんでした」エラーになり得る。
- 対応するなら、階生成時にマスター節点へ `weight/g` の並進質量
  （Ux, Uy）＋床の回転慣性（Rz, Σm·r²）を設定する配線が候補。ただし
  密度による自重との**二重計上防止**（DL には自重が含まれる）の設計が必要で、
  `standard-load-cases` の自重二重計上防止と同種の考慮を要する。
