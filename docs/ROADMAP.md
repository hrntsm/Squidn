## Phase 1: GUI 単独で「モデル作成→線形解析→検定」導線を完結させる

**ゴール**: 起動→節点/部材/断面/材料/境界条件/荷重を全て GUI で入力→線形静的解析→検定比表示、が一度も外部ファイルなしで可能。

- **1-1. `squid-n-edit` にコマンド追加**: `AddMaterial` / `DeleteMaterial` / `AddLoadCase` / `DeleteLoadCase` / (必要なら) `SetMaterialField`。undo/redo 対応・既存テストパターン踏襲
- **1-2. 材料 UI**: 材料タブに追加フォーム(名称・E・ポアソン比・密度・Fc/Fy)+プリセット(SN400B, SS400, Fc21 等)+削除。部材テーブル/インスペクタに材料割当コンボ追加
- **1-3. 荷重ケース UI**: 荷重タブに「ケース追加/削除」。新規モデルでも荷重入力に到達できることを確認
- **1-4. 部材・断面の削除ボタン**(使用中チェック付き)
- **1-5. 新規モデルテンプレート**: 起動時 or「新規」で最小サンプル(門型ラーメン)を1クリック生成するボタン — 導線の自己検証にもなる

## Phase 2: プロジェクト保存/読込(SCZ)の接続

- **2-1. ファイルメニュー**: 新規/開く/保存/名前を付けて保存。`rfd` によるネイティブダイアログ。保存先パス保持・タイトルバー表示・未保存変更マーカー
- **2-2. undo スタックと staleness の整合**(読込時リセット)

## Phase 3: ソルバー安定化

- **3-1. 特異系診断**: factorize 失敗時に拘束不足の候補(拘束ゼロの剛体モード方向、孤立節点、材料/断面未割当部材)を検出してエラーメッセージに含める。`Model::validate` の解析前実行
- **3-2. `DirectSparseLu` の `todo!` 撤去**(未実装エラー化 or 実装)
- **3-3. 固有値: 非収束をエラー/警告として返す**。質量ゼロ検出
- **3-4. 地震静的: stories なし/重量ゼロを明示エラー化**
- **3-5. analysis.rs の重複排除**(変位展開・断面力復元の共通化)
- **3-6. (調査結果を受けて追記)**

## Phase 4: 階定義と地震系解析の導線

- **4-1. 階の自動生成**: 節点 Z 座標のクラスタリングから stories を自動生成し、剛床(diaphragm)と地震重量(荷重ケースから換算 or 手入力)を設定する UI
- **4-2. 解析タブ拡充**: 地震方向 X/Y、AiMode、Z・地盤種別・C0 入力、固有値モード数入力
- **4-3. プッシュオーバー配線**: 設定(方向・ステップ・目標変位)+実行+性能曲線表示
- **4-4. 時刻歴配線**: 地震波 CSV 読込+Newmark 設定+減衰設定+実行、`time_history_view` を実データ接続(ダミー撤去)

## Phase 5: 設計(二次設計)とレポート

- **5-1. 検定→二次設計表示**: 層間変形角・剛性率・偏心率の層テーブル表示
- **5-2. 設計タブから NG 部材の 3D ハイライト連動**
- **5-3. レポートタブ**: 最低限の CSV/テキスト出力(モデル諸元・解析結果・検定表)

## Phase 6: 品質ゲート

- **6-1. `cargo test --workspace` 全緑 / clippy -D warnings / fmt**
- **6-2. 導線の統合テスト**(App レベル: 空→モデル構築→解析→検定が Rust テストで通る)
- **6-3. 申し送り文書** `docs/申し送り.md` 作成
