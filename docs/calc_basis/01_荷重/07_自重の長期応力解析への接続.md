# 1.7 自重の長期応力解析への接続（DL 荷重ケース）

部材自重・壁自重・ダンパー自重を、固定荷重（`LoadCaseKind::Dead`）の標準荷重ケース「DL」の内容として自動生成し（スラブ固定荷重の分配と合算）、長期応力解析・荷重組合せへ接続する。
梁は自重による等分布荷重（梁自重による CMQ に相当）、柱は軸力、壁・シェルは頂点への節点荷重（三方スリット壁は上部大梁側の節点へ全量伝達）となる。

**算定式**：総重量は §1.6 の自重算定規則（`enumerate_self_weight`）と同一とし、線材は次式の等分布荷重、その他は節点荷重に変換する（総量保存）。

\\[ w = \text{総重量}/\text{節点間長} \\]

**実装**：`squid_n_load::self_weight::self_weight_case_content`（`self_weight.rs`）が生成し、アプリの解析実行系（`sync_gravity_load_cases_action`）がスラブ分配と合算して「DL」へ冪等に同期する。
地震用重量の集計では DL（自重を含む）を重力ケースとして算入し、階の自動生成では密度からの自重直接算入を無効にする（`generate_stories_with_opts` の `include_density_self_weight = false`。両方行うと二重計上になる）。DL ケースが無い旧構成のモデルでは従来どおり密度から直接算入する。

**旧スキーマとの互換**：旧来の自動生成ケース「自重(自動)」「床荷重(自動)」は、モデル読込時に「DL」へ移行（改名・統合）される（`Model::migrate_legacy_auto_load_cases`）。
