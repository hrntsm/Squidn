# UI 導線改善（実装済み機能と UI の接続）

作成日: 2026-07-05
対象ブランチ: `claude/ui-navigation-flow-dgwe2j`
関連文書: `docs/申し送り.md`（残課題 3・4 に対応）、`specs/UI設計.md`（§1.1 双方向連動・§2 反復動線）

## 1. 調査で確認したギャップ

GUI (squid-n-app) から到達できない実装済み機能、および導線が弱い箇所を全数調査した結果:

### 実装済みだが入口がゼロだった機能
| 機能 | 実装場所 | 状態 |
|---|---|---|
| ST-Bridge 2.0 入出力 | `squid-n-io::stbridge` (export/import とも実装済) | メニューに項目なし・参照 0 件 |
| 荷重組合せ | `squid-n-load::combo::auto_combinations` + `squid-n-solver::Analysis::linear_combination` + `Model.combinations` | 作成・実行 UI なし・参照 0 件 |
| 保有水平耐力判定・Ds | `squid-n-design-jp::holding_capacity::{check_holding_capacity, ds_value, qud_by_story}` | 設計タブ未統合・参照 0 件 |

### 導線が弱かった箇所
- 変形図が常に `statics.first()` を表示（ケースを切り替えても変形図が変わらない）。
- ナビゲータの「結果ケース」がラベル表示のみでクリック不可。
- 断面テーブルの行が選択不可（節点・部材・荷重ケースは選択→連動があるのに断面だけ非対応）。
- 設計タブの NG 部材から断面編集への遷移がない（UI設計 §2 の「反復動線」未達）。
- ナビゲータ「階/レベル」が「（未実装）」プレースホルダのまま（階自動生成は実装済みなのに）。
- ナビゲータ「部材グループ」がクリックしても何も絞り込まない偽フィルタ。
- 解析実行後の結果タブ自動遷移がプッシュオーバー・時刻歴のみ（線形静的・地震静的は遷移しない不統一）。

## 2. 導線設計と実装内容

### 2.1 ファイルメニュー → ST-Bridge 入出力
「📥 ST-Bridge 読込…」「📤 ST-Bridge 書出…」をファイルメニューに追加。
読込は validate 後にモデル差し替え（プロジェクトパスは None のまま＝SCZ とは別系統）。
ヘッドレステスト可能な `import_stbridge_from` / `export_stbridge_to` をロジック層に置き、
rfd ダイアログは gui feature 側の薄いラッパとした（既存の scz 保存/読込と同じ構造）。

### 2.2 荷重タブ → 荷重組合せの作成、解析タブ → 組合せ実行
- `squid-n-edit` に `AddCombination` / `DeleteCombination` を新設（undo 可、MCP と共通経路）。
- 荷重タブに「荷重組合せ」セクション: 一覧（名前・項の内訳）・削除・
  DL/LL/地震X/地震Y/積雪のケース割当てから `auto_combinations` で一括生成。
- 解析タブに「荷重組合せ」グループ: 組合せを選んで実行 → `linear_combination`。
- 結果は `ResultsBundle.combos: Vec<(String, StaticOnce)>` に名前付きで保持し、
  実行後は検定 (`run_design_check`) も更新する。

### 2.3 結果ケースの選択導線（変形図の表示対象）
- `App::current_static()` を導入: ナビゲータで選択した結果（静的ケース or 組合せ）→
  未選択なら最後に実行した結果、の順で表示対象を解決する。
- ナビゲータ「結果ケース」をクリック可能にし、変形図・層指標がクリックで切り替わる。
- 変形図の `statics.first()` 固定バグを `current_static()` に置換。

### 2.4 設計タブ → 保有水平耐力（ルート3）セクション
- 前提データの充足状態をガイド表示（階未定義→自動生成へ、プッシュオーバー未実行→解析タブへ、
  のジャンプボタン付き）。
- 架構種別（RC/S × ラーメン/壁・ブレース）と部材ランク（FA〜FD）を選択 → `ds_value` で Ds を決定
  （部材ランクの自動判定は未実装のため選択値による簡易運用であることを UI に明記）。
- Qud は `qud_by_story`（地震重量・Z・Rt・T1）、Fes/Rs/Re は既存の層指標計算を利用し、
  `check_holding_capacity` で Qu ≥ Qun = Ds·Fes·Qud を層別判定・色分け表示。
- あわせて squid-n-solver のプッシュオーバーが `CapacityPoint.story_shear`/`story_drift` を
  常に空 Vec で返していた欠落（Qu が常に 0 と判定される）を発見し、層せん断（層以上の
  水平内力の上層からの累積）と層間変位（剛床マスター変位差）の算定を実装した。

### 2.5 細かい導線の統一
- 断面テーブルの ID 列をクリック可能に（`focus_section` 連動）。断面選択時はインスペクタに
  断面詳細＋使用部材の 3D ハイライトボタンを表示。
- 設計タブ検定表に「断面」列を追加し、クリックでモデルタブ断面サブタブへジャンプ
  （NG 部材 → 断面編集の反復動線）。
- ナビゲータ「階/レベル」を実データ表示に置換（空なら解析タブへの誘導ボタン）。
- ナビゲータ「部材グループ」クリックで該当部材群を 3D ハイライト（偽フィルタ解消）。
- 線形静的・地震静的も成功時に結果タブへ自動遷移（プッシュオーバー・時刻歴と統一）。

## 3. 今回見送った項目（優先度順の申し送り）

1. **スラブ入力 UI + CMQ データ供給**: `distribute_slab` は実装済みだが `Model.slabs` の
   入力 UI から作る必要があり規模が大きい。CMQ 表示モードは案内文を正直な内容に修正のみ。
2. **解析の非同期化＋進捗表示**（P8）: 引き続き UI スレッド同期実行。
3. **時刻歴の Y 方向・Rayleigh 減衰・HHT-α の配線**: ソルバー実装済み・UI 未配線のまま。
4. **`EditSectionShape`（既存断面のパラメトリック再編集）の UI**: コマンドは実装済み・未配線のまま。
5. **部材ランクの自動判定**: `rc_member_rank`（Qsu/Qmu 要）・`s_member_rank`（幅厚比要）は
   入力データが揃わないため、Ds は選択値運用とした。
6. **階の手動編集**（階高・地震重量・所属節点）: 専用編集コマンドの新設が必要。
